### New proposed-pass1 vocabulary introduced by repro pass

The repro pass introduces three pass1 values that are not yet in the spec section 7 vocabulary; they are proposed here for spec amendment in the closeout:

-   `closed:not-reproducible-on-v4.10.0`: bug already fixed (or feature already shipped) on v4.10.0; close with explicit verdict citation rather than the generic stale-24mo template.
-   `closed:feature-implemented`: feature ask already shipped (separate from PR-resolved because no specific PR can be cited in the close comment, just a confirmation that the requested behaviour exists).
-   `closed:pr-resolved-#NNNN`: same as the existing spec value but reused for repro-discovered fixes (e.g. #3283 -> PR #3361, #3605 -> PR #3651). The PR was found via repro, not via GitHub linkage.
